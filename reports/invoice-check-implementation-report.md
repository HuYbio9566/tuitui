# 财务票据识别与核验独立工作台实现报告

日期：2026-07-13

## 产品定位

该工具采用异常驱动审核模式：系统批量完成 OCR、规则校验和报销材料匹配，财务人员只处理低置信度、金额不一致、疑似重复、缺少合同等异常，而不是逐张阅读全部票据。

## 独立交互结构

- 入口：报销材料核验、票据字段识别、合同/订单/发票匹配三种任务，以及待处理批次。
- 左侧队列：异常、待复核、已通过分组，显示供应商、发票号、金额和异常类型。
- 中间原票：票据预览、字段框定位、缩放、旋转和 OCR 置信度。
- 右侧核验：识别字段、企业规则、关联材料三类核对视图。
- 业务处理：人工确认字段、退回补充、标记例外、确认通过、重复票据对比和批次提交。

## 关键控制规则

- 重复票据不能直接确认通过，必须先核对历史入账记录。
- 人工确认一个低置信度字段时，不自动消除其他字段异常。
- 金额异常同时展示发票、报销单、支付凭证和项目预算，避免只做单据 OCR。
- 缺少合同等附件问题可直接向提交人发起补充请求，并记录截止时间。
- 批次完成度由真实票据状态和本次处理结果计算。
