# 合同/文档版本对比独立工作台实现报告

日期：2026-07-13

## 产品定位

该工具是语义 Diff 阅读器，而不是两份文档摘要。它同时呈现文字变化、责任变化和风险影响，并让审阅人逐项接受、拒绝或评论，最终生成有决策依据的合并版本。

## 独立交互结构

- 入口：基准版本、修订版本两个明确版本槽，可追加第三版做一对多比较。
- 左侧：修改、新增、删除、风险上升和已决策差异导航。
- 中间：并排对比与红线模式，显示版本、页码和同步滚动状态。
- 右侧：语义影响、责任变化、修改前后内容、AI 处理建议和人工决策。
- 业务动作：接受、拒绝、评论协作、生成变化摘要、导出红线版和生成合并建议。

## 关键控制规则

- 风险上升差异未决策时不能生成合并稿。
- 红线模式保留删除和新增内容，不用摘要替代真实修改。
- 接受、拒绝和评论分别形成独立状态，仅更新当前差异。
- 评论可同时通知法务、采购并创建协作任务。
- 合并稿根据已有决策生成，并保存到原云盘谈判目录。

## 验证结果

- `node --check version-compare.js`：通过。
- 六组测试文件合计 39 项全部通过。
- 1440x900 浏览器主流程通过：追加第三版、并排对比、红线模式、合并阻断、接受、拒绝、评论、变化摘要和生成合并稿。
- 红线模式通过：当前差异包含一个删除段、一个新增段及 AI 语义影响说明。
- 风险决策通过：三项风险上升分别形成接受、拒绝和评论状态后，v4 合并建议才可生成。
- 390x844 响应式验证通过：差异、文档、影响三视图可切换，页面无横向溢出。
